#  IoTWF Standardized Architecture (Seven-Layer IoT Model)

## 1) ที่มาของโมเดล

ในปี **2014** คณะกรรมการสถาปัตยกรรมของ **IoT World Forum (IoTWF)** ซึ่งนำโดย:
- Cisco
- IBM
- Rockwell Automation
- และองค์กรชั้นนำอื่น ๆ

ได้เผยแพร่ **แบบจำลองสถาปัตยกรรม IoT แบบ 7 ชั้น** เพื่อเป็นมาตรฐานกลางในการอธิบายระบบ IoT ตั้งแต่ระดับอุปกรณ์จนถึงระดับธุรกิจ

โมเดลนี้กลายเป็น “ภาษากลาง” ที่ใช้สื่อสารกันในวงการ IoT ทั่วโลก

#  2) ทำไมต้องมีโมเดลสถาปัตยกรรม IoT?

ก่อนปี 2014 โมเดล IoT มีหลายแบบ เช่น:
- edge computing models
- cloud-centric models
- data pipeline models

แต่ทุกแบบมีความซับซ้อนและไม่เป็นมาตรฐานเดียวกัน

IoTWF จึงทำให้โมเดลเหล่านี้:
- **เรียบง่ายขึ้น**
- **ชัดเจนขึ้น**
- **เป็นมาตรฐานกลาง**
- **แบ่งเป็นชั้นที่มีฟังก์ชันเฉพาะ**
- **มี security ครอบคลุมทุกชั้น**

ผลลัพธ์คือสถาปัตยกรรม IoT ที่เข้าใจง่ายและนำไปใช้จริงได้ในองค์กร
# 3) โครงสร้างสถาปัตยกรรม IoT แบบ 7 ชั้น

โมเดลแบ่งเป็น 7 ชั้นจากล่างขึ้นบน (จาก Edge → Center):

### **1. Physical Devices & Controllers**

อุปกรณ์จริง เช่น เซนเซอร์, แอคชูเอเตอร์, เครื่องจักร, edge nodes หน้าที่: เก็บข้อมูลจากโลกจริง

### **2. Connectivity**

เครือข่ายที่ใช้เชื่อมต่อ เช่น Wi‑Fi, BLE, ZigBee, LoRaWAN, Cellular หน้าที่: ส่งข้อมูลจากอุปกรณ์เข้าสู่ระบบ

### **3. Edge (Fog) Computing**

การประมวลผลใกล้แหล่งข้อมูล เช่น filtering, preprocessing หน้าที่: ลดภาระ cloud และลด latency

### **4. Data Accumulation**

การจัดเก็บข้อมูล เช่น databases, data lakes หน้าที่: เก็บข้อมูลที่ไหลเข้ามาอย่างต่อเนื่อง

### **5. Data Abstraction**

การจัดระเบียบข้อมูล เช่น aggregation, indexing, APIs หน้าที่: ทำให้ข้อมูลพร้อมใช้งานโดยระบบอื่น

### **6. Application**

ระบบที่ใช้ข้อมูล เช่น dashboard, analytics, automation หน้าที่: สร้างคุณค่าจากข้อมูล

### **7. Collaboration & Processes**

ระดับธุรกิจ เช่น workflow, decision-making, human interaction หน้าที่: นำข้อมูลไปใช้ในกระบวนการจริงขององค์กร

#  4) ความสำคัญของโมเดลนี้

###  ทำให้ภาพ IoT ชัดเจนขึ้น

จากเดิมที่ IoT ดูเป็นระบบกระจัดกระจาย โมเดลนี้ทำให้เห็นว่า:
- ข้อมูลเริ่มจากอุปกรณ์
- ผ่านเครือข่าย
- ถูกประมวลผล
- ถูกจัดเก็บ
- ถูกนำไปใช้
- และสุดท้ายถูกนำไปใช้ในกระบวนการธุรกิจ

###  แต่ละชั้นมีฟังก์ชันเฉพาะ
ช่วยให้การออกแบบระบบ IoT:
- เป็นระบบ
- แบ่งงานได้ชัดเจน
- เลือกเทคโนโลยีได้เหมาะสม
- แก้ปัญหาได้ตรงจุด

### มี security ครอบคลุมทุกชั้น

IoTWF ระบุชัดว่า **ทุกชั้นต้องมีมาตรการความปลอดภัย** เช่น:
- secure silicon
- secure network access
- secure communication
- secure storage
- authentication/authorization
- identity management

นี่คือแนวคิด “security by design” สำหรับ IoT

# 5) ทำไม IoTWF Model ถูกใช้เป็นมาตรฐานสอน IoT?

เพราะมัน:
- ครอบคลุมทุกส่วนของระบบ IoT
- เข้าใจง่าย
- ใช้ได้กับทุกอุตสาหกรรม
- รองรับ edge computing และ cloud computing
- รองรับการเติบโตของข้อมูล
- รองรับการขยายระบบในอนาคต
- มี security ครอบคลุมทุกชั้น
- เป็นโมเดลที่องค์กรใหญ่ใช้จริง

สำหรับนักเรียนและวิศวกรใหม่ โมเดลนี้ช่วยให้:
- มองภาพรวมของระบบ IoT ได้ทันที
- เข้าใจว่าแต่ละชั้นทำอะไร
- เข้าใจว่าต้องเลือกเทคโนโลยีอย่างไร
- เข้าใจว่าความปลอดภัยต้องอยู่ทุกชั้น

# สรุป IoTWF Standardized Architecture

| ชั้น                 | หน้าที่                |
| -------------------- | ---------------------- |
| 1. Physical Devices  | เก็บข้อมูลจากโลกจริง   |
| 2. Connectivity      | ส่งข้อมูลผ่านเครือข่าย |
| 3. Edge Computing    | ประมวลผลใกล้อุปกรณ์    |
| 4. Data Accumulation | จัดเก็บข้อมูล          |
| 5. Data Abstraction  | ทำให้ข้อมูลพร้อมใช้งาน |
| 6. Application       | วิเคราะห์/ควบคุม       |
| 7. Collaboration     | กระบวนการธุรกิจ        |
