แม้ IoT จะเป็นแนวคิดใหม่ในช่วงทศวรรษที่ผ่านมา แต่รากฐานของมันตั้งอยู่บนเทคโนโลยีหลายแขนงที่พัฒนามายาวนาน โดยเฉพาะเทคโนโลยีด้าน **Internet/Web**, **Network**, และ **Device abstraction** ซึ่งทำให้ IoT สามารถทำงานร่วมกับอุปกรณ์จำนวนมากได้อย่างมีมาตรฐาน
## 1) เทคโนโลยี Internet / Web

Internet เป็นพื้นฐานสำคัญที่สุดของ IoT เพราะ IoT คือการเชื่อมอุปกรณ์จำนวนมากเข้ากับโลกดิจิทัลผ่านเครือข่ายอินเทอร์เน็ต
### **Internet ทำงานบน protocol มาตรฐาน**

อินเทอร์เน็ตถูกออกแบบให้ใช้ **มาตรฐานกลาง** เช่น
- TCP
- UDP
- IP (IPv4/IPv6)
- HTTP/HTTPS
- DNS
- TLS

มาตรฐานเหล่านี้ทำให้ระบบสามารถสื่อสารกันได้แม้จะถูกสร้างขึ้นในยุคที่แตกต่างกัน หรือโดยผู้ผลิตที่แตกต่างกัน

### **เพื่อความ compatible กับทั้งอุปกรณ์ในอดีตและอนาคต**

Internet ถูกออกแบบให้รองรับอุปกรณ์ตั้งแต่ยุคคอมพิวเตอร์เครื่องใหญ่จนถึงอุปกรณ์ IoT ขนาดเล็กในปัจจุบัน ความเข้ากันได้ย้อนหลัง (backward compatibility) และความสามารถรองรับอนาคต (forward compatibility) คือหัวใจสำคัญที่ทำให้ IoT สามารถเติบโตได้อย่างรวดเร็ว
### **Internet มี services มาตรฐาน (ทำงานบน cloud)**

บริการบนอินเทอร์เน็ต เช่น
- Web service
- REST API
- MQTT broker
- Cloud storage
- Authentication service

ทำให้ IoT สามารถส่งข้อมูลขึ้น cloud, ประมวลผล, วิเคราะห์ และสั่งงานกลับไปยังอุปกรณ์ได้อย่างเป็นระบบ

---
## 📡 2) เทคโนโลยี Network

IoT ต้องรองรับอุปกรณ์จำนวนมากที่มีข้อจำกัดแตกต่างกัน เช่น พลังงานต่ำ ระยะไกล หรือแบนด์วิดท์จำกัด จึงต้องมี network stack ที่หลากหลาย
### **Network ทำงานบน stack ที่หลากหลาย**
ตัวอย่าง stack ที่ใช้ใน IoT ได้แก่
- Wi‑Fi → TCP/IP stack
- Bluetooth LE → GATT/GAP
- LoRaWAN → MAC + Network server
- NB‑IoT → Cellular stack
- Zigbee → IEEE 802.15.4 stack
- Ethernet → TCP/IP stack

แต่ละ stack ถูกออกแบบให้เหมาะกับงานเฉพาะ เช่น พลังงานต่ำ ระยะไกล หรือความเสถียรสูง

### **Physical device เดียวกันอาจทำงานได้บนหลาย logical device**

ในระบบเครือข่าย อุปกรณ์ทางกายภาพ (physical device) สามารถมีหลายบทบาทในเชิงตรรกะ (logical device)

ตัวอย่างที่ชัดเจนคือ **Network Interface Card (NIC)**
- เป็น physical device เพียงชิ้นเดียว
- แต่สามารถทำงานได้ทั้ง **IPv4** และ **IPv6**
- สามารถมีหลาย MAC address (เช่น virtual NIC)
- สามารถรองรับหลาย protocol พร้อมกัน เช่น TCP, UDP, ICMP
แนวคิดนี้ทำให้ IoT สามารถสร้างระบบที่ยืดหยุ่นและรองรับการขยายตัวได้ง่าย

## 🔧 3) Device Abstraction และ Virtualization

เพื่อให้ IoT ทำงานได้กับอุปกรณ์จำนวนมาก จำเป็นต้องมีแนวคิดที่แยก “อุปกรณ์จริง” ออกจาก “อุปกรณ์เชิงตรรกะ”
ตัวอย่างเช่น
- Virtual device บน cloud
- Digital twin
- Logical sensor/actuator
- Software-defined network (SDN)

สิ่งเหล่านี้ทำให้ IoT สามารถจัดการอุปกรณ์จำนวนมากได้อย่างเป็นระบบ โดยไม่ต้องผูกติดกับ hardware โดยตรง

### Networking standard & technology

เทคโนโลยีเครือข่ายเป็นรากฐานสำคัญของ IoT เพราะ IoT คือระบบที่ต้องพึ่งพาการสื่อสารระหว่างอุปกรณ์จำนวนมากที่มีข้อจำกัดแตกต่างกัน ทั้งด้านพลังงาน ระยะทาง ความเร็ว และความเสถียร การเข้าใจมาตรฐานเครือข่ายและเทคโนโลยีที่เกี่ยวข้องจึงเป็นสิ่งจำเป็นก่อนเข้าสู่ IoT โดยตรง

## 1) Networking Standards — มาตรฐานเครือข่าย

มาตรฐานเครือข่ายถูกสร้างขึ้นเพื่อให้ระบบต่าง ๆ สามารถสื่อสารกันได้อย่างเข้ากันได้ (interoperable) ไม่ว่าจะเป็นอุปกรณ์รุ่นเก่า อุปกรณ์ใหม่ หรืออุปกรณ์จากผู้ผลิตต่างกัน

![](Week-01/Images/Networking%20standard.png)

### **1.1 OSI Model — โมเดลอ้างอิง 7 ชั้น**
แม้ OSI จะไม่ถูกใช้ตรง ๆ ในระบบจริง แต่เป็นโมเดลที่ใช้เพื่อทำความเข้าใจการสื่อสารแบบเป็นชั้น (layered communication)
- Physical
- Data Link
- Network
- Transport
- Session
- Presentation
- Application

โมเดลนี้ช่วยให้ผู้เรียนเข้าใจว่าแต่ละชั้นมีหน้าที่อะไร และเทคโนโลยีใดอยู่ในแต่ละชั้น

### **1.2 TCP/IP Model — โมเดลที่ใช้งานจริง**
เป็นโมเดลที่อินเทอร์เน็ตใช้งานจริง มี 4 ชั้น
- Application
- Transport
- Internet
- Network Access

เทคโนโลยีที่ IoT ใช้จำนวนมาก เช่น HTTP, MQTT, TCP, UDP, IP, Wi‑Fi ล้วนอยู่ในโมเดลนี้

## 2) Internet Protocols — โปรโตคอลพื้นฐานของอินเทอร์เน็ต

IoT ใช้โปรโตคอลมาตรฐานของอินเทอร์เน็ตเพื่อให้รองรับอุปกรณ์จำนวนมากและทำงานร่วมกับระบบในอนาคตได้
### **2.1 IP (Internet Protocol)**
- IPv4 → ใช้แพร่หลายที่สุด
- IPv6 → รองรับอุปกรณ์จำนวนมหาศาลในยุค IoT

อุปกรณ์ IoT จำนวนมากจะต้องรองรับ IPv6 ในอนาคตเพื่อให้สามารถเชื่อมต่อได้อย่างยั่งยืน
### **2.2 Transport Protocols**
- TCP → เชื่อถือได้, มีการตรวจสอบ
- UDP → เบา, เร็ว, เหมาะกับอุปกรณ์พลังงานต่ำ

### **2.3 Application Protocols**
- HTTP/HTTPS → ใช้ใน Web API
- MQTT → โปรโตคอลหลักของ IoT
- CoAP → โปรโตคอลเบาสำหรับอุปกรณ์พลังงานต่ำ
- WebSocket → สำหรับ real‑time communication

## 3) Network Technologies — เทคโนโลยีเครือข่ายที่ IoT ใช้

IoT ต้องรองรับหลายรูปแบบของเครือข่ายตามข้อจำกัดของอุปกรณ์

### **3.1 Short‑range networks**
- Wi‑Fi (IEEE 802.11)
- Bluetooth / BLE
- Zigbee
- Thread

เหมาะกับงานภายในบ้านหรืออาคาร

### **3.2 Long‑range networks**

- LoRaWAN
- Sigfox
- NB‑IoT
- LTE‑M
- 4G/5G

เหมาะกับงานภาคสนาม เช่น เกษตรกรรม อุตสาหกรรม หรือเมืองอัจฉริยะ

### **3.3 Wired networks**

- Ethernet
- RS‑485
- Modbus

เหมาะกับงานอุตสาหกรรมที่ต้องการความเสถียรสูง

## 4) Physical vs Logical Device

แนวคิดสำคัญของระบบเครือข่ายคือการแยก “อุปกรณ์จริง” ออกจาก “อุปกรณ์เชิงตรรกะ”
### **Physical device เดียวกันอาจทำงานได้บนหลาย logical device**

ตัวอย่างเช่น **Network Interface Card (NIC)**

- เป็น hardware เพียงชิ้นเดียว
- แต่รองรับหลาย logical functions เช่น
    - IPv4
    - IPv6
    - Multiple MAC addresses
    - Virtual NIC
    - TCP/UDP stack พร้อมกัน

แนวคิดนี้ทำให้ IoT สามารถสร้างระบบที่ยืดหยุ่นและรองรับการขยายตัวได้ง่าย

## เทคโนโลยีการเข้าถึงเครือข่ายในระดับชั้น Physical

เทคโนโลยีเหล่านี้คือ “ช่องทางการสื่อสารจริง” ที่อุปกรณ์ IoT ใช้ในการส่งข้อมูลออกจากโลก physical ไปยังโลก digital โดยตรง แต่ละเทคโนโลยีถูกออกแบบมาเพื่อรองรับข้อจำกัดที่แตกต่างกัน เช่น ระยะทาง พลังงาน ความเร็ว และความเสถียร

รายการเทคโนโลยีที่เราจะอธิบายทีละข้อ ได้แก่:
- LPWAN
- Cellular
- Bluetooth Low Energy (BLE)
- ZigBee
- NFC
- RFID
- Wi‑Fi
- Ethernet

และเราจะเริ่มจาก **LPWAN** ซึ่งเป็นเทคโนโลยีสำคัญที่สุดในงาน IoT ภาคสนาม

#  LPWAN (Low Power Wide Area Network)


![](Week-01/Images/LPWAN.png)

## 1) แนวคิดหลักของ LPWAN

LPWAN คือเทคโนโลยีสื่อสารไร้สายที่ถูกออกแบบมาเพื่อรองรับอุปกรณ์ IoT ที่ต้องการ:
- **พลังงานต่ำมาก (low power)**
- **ระยะทางไกลมาก (long range)**
- **อัตราการส่งข้อมูลต่ำ (low data rate)**
- **ต้นทุนการใช้งานต่ำ (low cost)**
- **รองรับอุปกรณ์จำนวนมาก (massive IoT)**

LPWAN จึงเหมาะกับอุปกรณ์ประเภท **wireless sensors** ที่ต้องทำงานเป็นปี ๆ โดยใช้แบตเตอรี่ก้อนเล็ก เช่น เซนเซอร์วัดอุณหภูมิในฟาร์ม, เซนเซอร์วัดระดับน้ำ, เซนเซอร์ตรวจสภาพอากาศ, เซนเซอร์ในระบบเกษตรอัจฉริยะ ฯลฯ

## 2) คุณสมบัติเด่นของ LPWAN
- **พลังงานต่ำมาก** อุปกรณ์สามารถทำงานได้หลายปีโดยไม่ต้องเปลี่ยนแบตเตอรี่
- **ระยะทางไกลมาก** ตั้งแต่หลายกิโลเมตรจนถึงหลายสิบกิโลเมตร ขึ้นกับเทคโนโลยี
- **ความเร็วต่ำ** เหมาะกับข้อมูลขนาดเล็ก เช่น ค่าเซนเซอร์
- **รองรับอุปกรณ์จำนวนมาก** เหมาะกับงานที่มีอุปกรณ์หลายพันหรือหลายหมื่นตัว
- **ต้นทุนโครงสร้างพื้นฐานต่ำ** ใช้ gateway จำนวนน้อยครอบคลุมพื้นที่กว้าง

## 3) ตัวอย่างเทคโนโลยี LPWAN

### **3.1 LoRa (Long Range)**
- เป็น **physical layer protocol**
- ใช้เทคนิค Chirp Spread Spectrum
- ระยะทางไกลมาก (หลายกิโลเมตร)
- ใช้พลังงานต่ำ
- นิยมใช้ในงานเกษตร, เมืองอัจฉริยะ, ระบบตรวจวัดระยะไกล
- ทำงานร่วมกับ LoRaWAN (network layer)

### **3.2 SigFox**
- เครือข่ายที่ใช้โครงสร้างพื้นฐานของผู้ให้บริการ
- ส่งข้อมูลแบบ ultra‑narrowband
- พลังงานต่ำมาก
- เหมาะกับงานที่ส่งข้อมูลน้อยมาก เช่น แจ้งเตือนสถานะ

### **3.3 LTE‑M (LTE Cat‑M1)**
- ทำงานบนเครือข่าย cellular
- รองรับ mobility
- latency ต่ำกว่า NB‑IoT
- เหมาะกับอุปกรณ์ที่ต้องเคลื่อนที่ เช่น asset tracking

### **3.4 NB‑IoT (Narrow‑Band IoT)**
- ใช้คลื่นความถี่ของผู้ให้บริการโทรคมนาคม
- พลังงานต่ำมาก
- ระยะไกล
- เหมาะกับงานที่ต้องการความเสถียรสูง เช่น ระบบสาธารณูปโภค
### **3.5 Haystack**
- เป็นเทคโนโลยี LPWAN ที่เน้นการติดตามวัตถุ
- ใช้พลังงานต่ำและครอบคลุมพื้นที่กว้าง
- นิยมในงาน logistics และ asset tracking
# รุป LPWAN

LPWAN คือเทคโนโลยีที่ทำให้ IoT สามารถขยายตัวไปสู่พื้นที่กว้าง เช่น ฟาร์ม โรงงาน เมือง และระบบสาธารณูปโภค โดยไม่ต้องใช้พลังงานมากและไม่ต้องติดตั้งโครงสร้างพื้นฐานราคาแพง

LPWAN คือ “กระดูกสันหลัง” ของ IoT ภาคสนาม (field IoT)

# Cellular (เครือข่ายโทรศัพท์เคลื่อนที่)

![](Week-01/Images/Cellular.png)

## 1) แนวคิดหลักของ Cellular สำหรับ IoT

Cellular คือเทคโนโลยีสื่อสารไร้สายที่ใช้โครงสร้างพื้นฐานของเครือข่ายโทรศัพท์เคลื่อนที่ เช่น เสาสัญญาณ (cell site) และระบบ core network ของผู้ให้บริการโทรคมนาคม
จุดเด่นของ Cellular คือ:
- **ครอบคลุมพื้นที่กว้างมาก** (ขึ้นกับ cell site)
- **ความเสถียรสูง**
- **รองรับ mobility** (อุปกรณ์เคลื่อนที่ได้ เช่น รถ, คน, สินค้า)
- **รองรับจำนวนอุปกรณ์มาก**
- **มีระบบความปลอดภัยระดับโทรคมนาคม**

Cellular จึงเหมาะกับงาน IoT ที่ต้องการความน่าเชื่อถือสูง เช่น ระบบติดตามทรัพย์สิน, ระบบสาธารณูปโภค, ระบบอุตสาหกรรม และระบบเมืองอัจฉริยะ

## 2) Cellular กับ IoT: LTE‑M และ NB‑IoT

ในยุค IoT มีเทคโนโลยีที่ถูกออกแบบเฉพาะสำหรับอุปกรณ์พลังงานต่ำ แต่ใช้โครงสร้างพื้นฐาน cellular ได้แก่:

### **LTE‑M (LTE Cat‑M1)**
- รองรับ mobility
- latency ต่ำ
- เหมาะกับอุปกรณ์ที่ต้องเคลื่อนที่ เช่น asset tracking, vehicle telematics
- ใช้พลังงานต่ำกว่า LTE ปกติ

### **NB‑IoT (Narrow‑Band IoT)**
- ใช้แบนด์วิดท์แคบมาก
- พลังงานต่ำมาก
- ระยะไกล
- เหมาะกับงานที่ต้องการความเสถียร เช่น ระบบน้ำ, ระบบไฟฟ้า, ระบบวัดมิเตอร์

ทั้งสองเทคโนโลยีนี้ถือเป็น **long‑range communication** ที่ใช้ cellular เป็นสื่อกลาง และเป็นส่วนหนึ่งของกลุ่มเทคโนโลยี LPWAN ที่ทำงานบนเครือข่ายโทรคมนาคม

## 3) ตัวอย่างเทคโนโลยี Cellular ตามยุคสมัย
### **2G (GSM)**
- ใช้กันอย่างแพร่หลายในอดีต
- รองรับการส่งข้อมูลแบบ GPRS/EDGE
- ปัจจุบันอยู่ในสถานะเตรียมเลิกให้บริการในหลายประเทศ
- อุปกรณ์ IoT รุ่นเก่าเคยใช้จำนวนมาก

### **CDMA (Code‑Division Multiple Access)**
- เป็นเทคโนโลยีคู่แข่งของ GSM ในอดีต
- ปัจจุบันอยู่ในสถานะเตรียมเลิกให้บริการ
- ใช้ในบางประเทศก่อนยุค 3G/4G

### **3G (WCDMA / HSPA)**
- รองรับการส่งข้อมูลความเร็วสูงกว่า 2G
- ปัจจุบันอยู่ในสถานะเตรียมเลิกให้บริการในหลายประเทศ
- อุปกรณ์ IoT รุ่นกลางเคยใช้

### **4G (LTE)**
- เป็นระบบหลักในปัจจุบัน
- รองรับการส่งข้อมูลความเร็วสูง
- ใช้ในงาน IoT ที่ต้องการแบนด์วิดท์มาก เช่น กล้องวงจรปิด
- จะถูกลดบทบาทเมื่อ 5G ครอบคลุมมากขึ้น
### **5G**
- ความเร็วสูงมาก
- latency ต่ำมาก
- รองรับ massive IoT (อุปกรณ์จำนวนมหาศาล)
- รองรับ URLLC (Ultra‑Reliable Low‑Latency Communication) สำหรับงานอุตสาหกรรม
- เป็นเทคโนโลยีหลักของ IoT ยุคอนาคต

#  สรุป Cellular สำหรับ IoT

Cellular เป็นเทคโนโลยีที่ทำให้ IoT สามารถทำงานได้ในพื้นที่กว้างและมีความเสถียรสูง โดยเฉพาะงานที่ต้องการความน่าเชื่อถือ เช่น ระบบติดตามทรัพย์สิน ระบบสาธารณูปโภค และระบบอุตสาหกรรม

| เทคโนโลยี | สถานะ         | ความเหมาะสมกับ IoT          |
| --------- | ------------- | --------------------------- |
| 2G / CDMA | เตรียมเลิกใช้ | IoT รุ่นเก่า                |
| 3G        | เตรียมเลิกใช้ | IoT รุ่นกลาง                |
| 4G        | ระบบหลัก      | IoT ที่ต้องการแบนด์วิดท์สูง |
| LTE‑M     | ใช้งานจริง    | IoT เคลื่อนที่ พลังงานต่ำ   |
| NB‑IoT    | ใช้งานจริง    | IoT พลังงานต่ำ ระยะไกล      |
| 5G        | กำลังขยาย     | Massive IoT, อุตสาหกรรม     |

# Bluetooth Low Energy (BLE)

![](Week-01/Images/BLE.png)

## 1) แนวคิดหลักของ BLE

Bluetooth Low Energy (BLE) เป็นเทคโนโลยีสื่อสารไร้สายระยะใกล้ที่ถูกออกแบบมาเพื่อรองรับอุปกรณ์ IoT ที่ต้องการพลังงานต่ำและส่งข้อมูลไม่มาก โดยเน้นการสื่อสารแบบเป็นช่วง ๆ (bursts) มากกว่าการส่งข้อมูลต่อเนื่อง

คุณสมบัติหลักของ BLE ได้แก่:
- **ระยะใกล้** (โดยทั่วไปไม่เกิน 100 เมตร)
- **พลังงานต่ำมาก**
- **Bandwidth ประมาณ 270 kbps**
- **เหมาะกับข้อมูลขนาดเล็กและไม่ต่อเนื่อง**
- **ต้นทุนต่ำและมีในอุปกรณ์พกพาจำนวนมาก**

BLE จึงเป็นเทคโนโลยีหลักในงาน IoT ที่เกี่ยวข้องกับอุปกรณ์สวมใส่ อุปกรณ์สุขภาพ และอุปกรณ์ที่ต้องการเชื่อมต่อกับสมาร์ตโฟน
## 2) Topology ของ BLE
BLE ใช้ topology แบบ **star** โดยมีอุปกรณ์สองบทบาท:
- **Primary (Central)** → อุปกรณ์หลัก เช่น สมาร์ตโฟน
- **Secondary (Peripheral)** → อุปกรณ์ลูก เช่น เซนเซอร์, อุปกรณ์สวมใส่

Primary หนึ่งตัวสามารถเชื่อมต่อกับ Secondary หลายตัวได้ ทำให้ BLE เหมาะกับงานที่มีอุปกรณ์จำนวนหนึ่งเชื่อมต่อกับศูนย์กลาง เช่น ระบบสุขภาพ ระบบติดตามกิจกรรม หรือระบบควบคุมอุปกรณ์ในบ้าน

## 3) BLE ใน OSI Model

BLE ทำงานใน **2 ชั้น** ของ OSI Model นั่นคือ
### **Layer 1: PHY (Physical Layer)**
- กำหนดรูปแบบสัญญาณวิทยุ
- ความถี่ 2.4 GHz
- กำหนด modulation และ channel spacing

### **Layer 2: MAC (Medium Access Control)**
- กำหนดวิธีการเข้าถึงช่องสัญญาณ
- กำหนดการจัดการ connection, advertising, scanning
- จัดการการสื่อสารระหว่าง central และ peripheral

BLE จึงเป็นเทคโนโลยีที่อยู่ในระดับชั้น physical และ data link ของ OSI Model โดยตรง

## 4) ลักษณะการใช้งาน BLE
BLE เหมาะกับงานที่
- ส่งข้อมูลไม่มาก
- ส่งข้อมูลเป็นช่วง ๆ (bursts)
- ต้องการพลังงานต่ำ
- ต้องการเชื่อมต่อกับสมาร์ตโฟนหรืออุปกรณ์พกพา
- ต้องการ latency ต่ำในระยะใกล้

ตัวอย่างข้อมูลที่ BLE มักส่ง เช่น ค่าเซนเซอร์สุขภาพ, สถานะอุปกรณ์, ข้อมูลการเคลื่อนไหว, ข้อมูลการแจ้งเตือนระยะใกล้

## 5) ตัวอย่างเทคโนโลยี BLE

### **หูฟังไร้สาย (Wireless Earbuds)**

แม้หูฟังจะใช้ Bluetooth Classic สำหรับเสียง แต่ BLE ถูกใช้สำหรับฟังก์ชันควบคุม เช่น pairing, battery status, sensor data
### **อุปกรณ์สวมใส่เพื่อสุขภาพ (Wearable Health Devices)**
เช่น
- สมาร์ตวอทช์
- ฟิตเนสแทร็กเกอร์
- เซนเซอร์วัดหัวใจ
- เซนเซอร์วัดการเคลื่อนไหว
อุปกรณ์เหล่านี้ต้องการพลังงานต่ำและส่งข้อมูลเป็นช่วง ๆ ซึ่งตรงกับคุณสมบัติของ BLE

#  สรุป BLE

BLE เป็นเทคโนโลยีสื่อสารไร้สายระยะใกล้ที่เหมาะกับอุปกรณ์ IoT พลังงานต่ำ โดยเฉพาะอุปกรณ์ที่ต้องการเชื่อมต่อกับสมาร์ตโฟนหรืออุปกรณ์พกพา และต้องการส่งข้อมูลไม่มากในรูปแบบ bursts

| คุณสมบัติ  | รายละเอียด                                  |
| ---------- | ------------------------------------------- |
| ระยะทาง    | ไม่เกิน 100 เมตร                            |
| Bandwidth  | ~270 kbps                                   |
| Topology   | Star (Central ↔ Peripheral)                 |
| OSI Layers | PHY + MAC                                   |
| เหมาะกับ   | Wearables, health devices, sensors ระยะใกล้ |

#  ZigBee

![](Week-01/Images/Zigbee.png)

## 1) แนวคิดหลักของ ZigBee

ZigBee เป็นเทคโนโลยีสื่อสารไร้สายที่ออกแบบมาเพื่อรองรับงาน IoT ภายในบ้านและอาคาร (Home Automation) โดยเน้นพลังงานต่ำ ความเสถียรสูง และการสื่อสารแบบกระจายตัว (distributed communication)

คุณสมบัติหลักของ ZigBee ได้แก่:
- ทำงานบนความถี่ **2.4 GHz**
- ระยะทางสื่อสาร
    - **ไม่เกิน 100 เมตรในที่โล่ง**
    - **ประมาณ 10–20 เมตรภายในอาคาร**
- Bandwidth ประมาณ **250 kbps**
- ใช้พลังงานต่ำ
- รองรับ topology แบบ **mesh**
- อุปกรณ์สามารถทำงานได้หลายบทบาท (device, controller, router)

ZigBee จึงเหมาะกับงานที่ต้องการความเสถียรในพื้นที่จำกัด เช่น บ้าน อาคาร สำนักงาน และระบบควบคุมอุปกรณ์ไฟฟ้า

## 2) ความถี่และระยะทาง

ZigBee ใช้ความถี่ **2.4 GHz** เช่นเดียวกับ Wi‑Fi และ BLE แต่ถูกออกแบบให้ใช้พลังงานต่ำกว่าและมีรูปแบบการสื่อสารที่เหมาะกับอุปกรณ์ IoT ขนาดเล็ก
ระยะทางโดยประมาณ:
- **พื้นที่โล่ง:** ~100 เมตร
- **ภายในอาคาร:** 10–20 เมตร (ขึ้นกับผนังและสิ่งกีดขวาง)

## 3) Bandwidth
ZigBee มี Bandwidth ประมาณ **250 kbps** เพียงพอสำหรับข้อมูลขนาดเล็ก เช่น:
- ค่าเซนเซอร์
- สถานะอุปกรณ์
- คำสั่งควบคุม
- ข้อมูลแจ้งเตือน

ไม่เหมาะกับข้อมูลขนาดใหญ่ เช่น วิดีโอหรือเสียง

## 4) Topology แบบ Mesh

จุดเด่นที่สุดของ ZigBee คือ topology แบบ **mesh network**
### **Mesh network คืออะไร?**
- อุปกรณ์ทุกตัวสามารถสื่อสารกันได้
- ข้อมูลสามารถวิ่งผ่านหลายเส้นทาง
- หากอุปกรณ์บางตัวดับหรือหลุดจากเครือข่าย ข้อมูลยังสามารถวิ่งผ่านเส้นทางอื่นได้
- ทำให้เครือข่ายมีความเสถียรสูงและครอบคลุมพื้นที่กว้างขึ้น

### **บทบาทของอุปกรณ์ ZigBee**
อุปกรณ์ ZigBee สามารถทำงานได้หลายหน้าที่ เช่น:
- **End Device** → อุปกรณ์ทั่วไป เช่น เซนเซอร์
- **Router** → ช่วยส่งต่อข้อมูลใน mesh
- **Coordinator** → อุปกรณ์ศูนย์กลางของเครือข่าย ZigBee
ความสามารถนี้ทำให้ ZigBee เหมาะกับระบบที่ต้องการอุปกรณ์จำนวนมากเชื่อมต่อกันอย่างต่อเนื่อง

## 5) การใช้งาน ZigBee ในงานจริง

ZigBee ถูกออกแบบมาเพื่อใช้งานใน **Home Automation** เช่น:
- ระบบไฟอัจฉริยะ (Smart Lighting)
- ปลั๊กไฟอัจฉริยะ
- เซนเซอร์ประตู/หน้าต่าง
- เซนเซอร์ตรวจจับความเคลื่อนไหว
- ระบบควบคุมอุณหภูมิ
- ระบบควบคุมอุปกรณ์ไฟฟ้าในบ้าน
- ระบบ Smart Home ที่เชื่อมต่อกับ Hub เช่น Philips Hue, Xiaomi, Aqara

จุดเด่นคือความเสถียร ความปลอดภัย และความสามารถในการสร้างเครือข่าย mesh ที่ครอบคลุมพื้นที่บ้านได้ดี

#  สรุป ZigBee

ZigBee เป็นเทคโนโลยีสื่อสารไร้สายที่เหมาะกับงาน IoT ภายในบ้านและอาคาร โดยเน้นพลังงานต่ำ ความเสถียรสูง และ topology แบบ mesh ที่ช่วยให้เครือข่ายมีความยืดหยุ่นและครอบคลุมพื้นที่ได้ดี

| Topology     | Mesh                                |
| บทบาทอุปกรณ์ | End Device, Router, Coordinator     |

#  NFC (Near Field Communication)

![](Week-01/Images/NFC.png)

## 1) แนวคิดหลักของ NFC

NFC เป็นเทคโนโลยีสื่อสารไร้สายระยะสั้นมาก (ultra‑short range) ที่ออกแบบมาเพื่อการสื่อสารระหว่างอุปกรณ์ที่อยู่ใกล้กันมาก ๆ โดยเน้นความปลอดภัย ความสะดวก และการใช้งานแบบแตะ (tap‑to‑interact)
คุณสมบัติหลักของ NFC ได้แก่
- **ระยะสื่อสารสั้นมาก** ประมาณ **4 เซนติเมตร**
- **พลังงานต่ำ**
- **ความปลอดภัยสูง** (เพราะต้องอยู่ใกล้มาก)
- **ใช้งานง่าย** เพียงแค่แตะหรือวางใกล้กัน
- **เหมาะกับข้อมูลขนาดเล็ก** เช่น ID, token, payment data

NFC จึงถูกใช้ในงานที่ต้องการความปลอดภัยและการยืนยันตัวตนแบบรวดเร็ว

## 2) รูปแบบอุปกรณ์ NFC

อุปกรณ์ NFC ส่วนใหญ่มีลักษณะดังนี้
### **Tag**
- เป็นชิปขนาดเล็ก
- ไม่ต้องใช้พลังงานจากแบตเตอรี่ (passive)
- เก็บข้อมูลเล็กน้อย เช่น ID, URL, token
- ใช้ในงาน asset tracking, check‑in, automation

### **Card**
- ใช้ในระบบชำระเงิน เช่นบัตรเครดิตแบบ contactless
- ใช้ในระบบขนส่ง เช่นบัตรโดยสาร
- ใช้ในระบบเข้าอาคาร เช่นบัตรพนักงาน

### **Device (Active NFC)**
- สมาร์ตโฟน
- เครื่องอ่าน (NFC reader)
- อุปกรณ์ที่สามารถจ่ายพลังงานให้ tag

## 3) ระยะทางสื่อสาร

NFC ถูกออกแบบให้มีระยะสื่อสาร **สั้นมาก** เพื่อความปลอดภัยและความแม่นยำในการยืนยันตัวตน:
- ระยะทั่วไป  **~4 เซนติเมตร**
- ระยะสูงสุดตามมาตรฐาน  **ไม่เกิน 10 เซนติเมตร**

ระยะที่สั้นทำให้การดักฟังหรือโจมตีทำได้ยาก จึงเหมาะกับงานด้านความปลอดภัย

## 4) ลักษณะการใช้งาน NFC

NFC เหมาะกับงานที่ต้องการ
- การยืนยันตัวตน
- การชำระเงิน
- การเช็คอิน
- การระบุตัวตนของวัตถุ
- การอ่านข้อมูลจาก tag
- การจับคู่ (pairing) อุปกรณ์อย่างรวดเร็ว

NFC ไม่เหมาะกับงานที่ต้องส่งข้อมูลจำนวนมากหรือสื่อสารระยะไกล

## 5) ตัวอย่างการใช้งาน NFC ในโลกจริง
### **ระบบ Payment**
- บัตรเครดิตแบบ contactless
- การชำระเงินผ่านสมาร์ตโฟน (Apple Pay, Google Pay)
- ระบบชำระเงินในร้านค้า

### **ระบบ Check‑in**
- เข้าอาคาร
- ลงทะเบียนเข้าร่วมกิจกรรม
- ระบบบันทึกเวลาเข้า–ออกงาน

### **Asset Tracking**
- ติด tag บนสินค้า
- ติด tag บนอุปกรณ์ในโรงงาน
- ใช้ตรวจสอบสถานะหรือข้อมูลของวัตถุ

#  สรุป NFC

NFC เป็นเทคโนโลยีสื่อสารไร้สายระยะสั้นมากที่เน้นความปลอดภัยและความสะดวกในการใช้งาน เหมาะกับงานที่ต้องการยืนยันตัวตนหรืออ่านข้อมูลจาก tag/card ในระยะใกล้มาก

| คุณสมบัติ     | รายละเอียด                        |
| ------------- | --------------------------------- |
| ระยะทาง       | ~4 เซนติเมตร                      |
| รูปแบบอุปกรณ์ | Tag, Card, Active Device          |
| Bandwidth     | ต่ำ (ข้อมูลขนาดเล็ก)              |
| จุดเด่น       | ปลอดภัย, ใช้งานง่าย               |
| เหมาะกับ      | Payment, Check‑in, Asset tracking |

#  RFID (Radio Frequency Identification)

![](นโลยีที่ใช้คลื่นวิทยุในการระบุตัวตนของวัตถุ โดยอาศัย **RFID tag** ที่เก็บข้อมูล (ส่วนใหญ่เป็น ID) และ **RFID reader** ที่ทำหน้าที่อ่านข้อมูลจาก tag
คุณสมบัติหลักของ RFID ได้แก่:
- ใช้คลื่นวิทยุในการสื่อสาร
- ระยะสื่อสารสั้น (ประมาณ **1 เมตร**)
- ส่งข้อมูลขนาดเล็ก เช่น ID หรือข้อมูลระบุตัวตน
- ใช้งานง่าย เพียงนำ tag เข้าใกล้ reader
- มีทั้งแบบ **active** และ **passive**

RFID เป็นเทคโนโลยีที่นิยมใช้ในงานระบุตัวตนของวัตถุ เช่น สินค้า อุปกรณ์ เครื่องมือ หรือบัตรเข้าอาคาร

## 2) RFID Tags ทำหน้าที่เก็บข้อมูล (ID)

RFID tag ส่วนใหญ่เก็บข้อมูลประเภท:
- หมายเลขประจำตัว (ID)
- หมายเลขสินค้า
- หมายเลขอุปกรณ์
- Token หรือข้อมูลระบุตัวตนอื่น ๆ

ข้อมูลใน tag มักมีขนาดเล็กและถูกอ่านโดย RFID reader ผ่านคลื่นวิทยุ

## 3) ระยะทางสื่อสาร

RFID ถูกออกแบบมาเพื่อใช้งานในระยะสั้น โดยทั่วไปประมาณ:
- **~1 เมตร** สำหรับระบบทั่วไป
- ระยะอาจมากขึ้นในระบบ active หรือระบบเฉพาะทาง

ระยะที่สั้นทำให้ RFID เหมาะกับงานที่ต้องการความแม่นยำในการอ่านข้อมูล เช่น การตรวจสินค้า การเช็คอิน หรือการระบุตัวตนของอุปกรณ์

## 4) ประเภทของ RFID Tag

RFID tag มีสองประเภทหลัก
### **Passive RFID Tag**
- **ไม่ต้องใช้แบตเตอรี่**
- ใช้พลังงานจากคลื่นวิทยุที่ส่งมาจาก reader
- ราคาถูก
- อายุการใช้งานยาวนาน
- ระยะสื่อสารสั้นกว่า active
- นิยมใช้มากที่สุด

### **Active RFID Tag**
- **มีแบตเตอรี่ในตัว**
- ส่งสัญญาณเองได้
- ระยะสื่อสารไกลกว่า passive
- ราคาสูงกว่า
- ใช้ในงานที่ต้องการระยะไกลหรือการติดตามแบบ real‑time
    

## 5) ความสัมพันธ์ระหว่าง RFID และ NFC
RFID และ NFC มีลักษณะการใช้งานคล้ายกัน เช่น
- ใช้ tag/card
- ใช้คลื่นวิทยุ
- ใช้สำหรับระบุตัวตน
- ใช้ในงาน check‑in หรือ asset tracking

แต่มีความแตกต่างสำคัญ:

| เทคโนโลยี | ระยะทาง      | ความสามารถ                                      |
| --------- | ------------ | ----------------------------------------------- |
| **RFID**  | ~1 เมตร      | อ่าน ID, ใช้ในคลังสินค้า, asset tracking        |
| **NFC**   | ~4 เซนติเมตร | ใช้ใน payment, check‑in, smartphone interaction |

# Wi‑Fi

![](Week-01/Images/WiFi.png)

## 1) แนวคิดหลักของ Wi‑Fi

Wi‑Fi เป็นเทคโนโลยีสื่อสารไร้สายที่ใช้คลื่นวิทยุในการเชื่อมต่ออุปกรณ์เข้ากับเครือข่ายคอมพิวเตอร์ โดยเฉพาะเครือข่ายอินเทอร์เน็ตภายในบ้าน อาคาร หรือสำนักงาน
คุณสมบัติหลักของ Wi‑Fi ได้แก่:
- ใช้คลื่นวิทยุในย่าน **2.4 GHz** และ **5 GHz**
- รองรับอัตราการส่งข้อมูลสูง
- ระยะสื่อสารปานกลาง (ภายในอาคาร)
- ใช้พลังงานมากกว่า BLE, ZigBee, LPWAN
- เหมาะกับอุปกรณ์ที่ต้องการแบนด์วิดท์สูง เช่น กล้อง, อุปกรณ์สมาร์ตโฮมบางประเภท

## 2) มาตรฐาน IEEE 802.11

Wi‑Fi ทำงานบนข้อกำหนดของมาตรฐาน **IEEE 802.11** ซึ่งมีหลายรุ่น เช่น:
- **802.11a**
- **802.11b**
- **802.11g**
- **802.11n**

แต่ละรุ่นมีความเร็วและการใช้พลังงานแตกต่างกัน

### **802.11b**
- ความเร็วสูงสุด ~11 Mbps
- ใช้พลังงานต่ำกว่า /n
- นิยมในอุปกรณ์ IoT

### **802.11g**
- ความเร็วสูงสุด ~54 Mbps
- ใช้พลังงานปานกลาง
- นิยมในอุปกรณ์ IoT เช่นเดียวกับ /b

### **802.11n**
- ความเร็วสูงสุดหลายร้อย Mbps
- รองรับ MIMO
- **กินพลังงานมากที่สุด**
- เหมาะกับอุปกรณ์ที่ต้องการแบนด์วิดท์สูง เช่น กล้องวงจรปิด

## 3) การเลือกใช้ Wi‑Fi ในระบบ IoT

แม้ Wi‑Fi จะมีความเร็วสูง แต่ก็มีข้อจำกัดสำคัญสำหรับ IoT:
- **ใช้พลังงานมาก**
- **ระยะสื่อสารไม่ไกลมาก**
- **ต้องมีโครงสร้างพื้นฐาน (Access Point)**
- **ไม่เหมาะกับอุปกรณ์พลังงานต่ำที่ต้องทำงานหลายปี**

ดังนั้นอุปกรณ์ IoT ส่วนใหญ่จะเลือกใช้:
- **802.11b** หรือ **802.11g** เพื่อประหยัดพลังงานและรองรับการสื่อสารพื้นฐาน
## 4) แนวโน้มในอนาคต

มีแนวโน้มว่า Wi‑Fi จะถูกแทนที่ด้วยเทคโนโลยีที่ใช้พลังงานต่ำกว่า เช่น:
- BLE
- ZigBee
- Thread
- LoRaWAN
- NB‑IoT
- LTE‑M

โดยเฉพาะในงานที่ต้องการพลังงานต่ำและอุปกรณ์จำนวนมาก เช่น เซนเซอร์ในบ้านหรืออาคาร

อย่างไรก็ตาม Wi‑Fi ยังคงมีบทบาทสำคัญในงานที่ต้องการแบนด์วิดท์สูง เช่น:
- กล้องวงจรปิด
- อุปกรณ์สมาร์ตโฮมที่ต้องการสตรีมข้อมูล
- อุปกรณ์ที่ต้องเชื่อมต่ออินเทอร์เน็ตโดยตรง

# สรุป Wi‑Fi

Wi‑Fi เป็นเทคโนโลยีสื่อสารไร้สายที่ใช้คลื่นวิทยุและรองรับอัตราการส่งข้อมูลสูง แต่ใช้พลังงานมาก จึงเหมาะกับอุปกรณ์ IoT ที่ต้องการแบนด์วิดท์สูง ไม่ใช่อุปกรณ์พลังงานต่ำ

| คุณสมบัติ | รายละเอียด                                |
| --------- | ----------------------------------------- |
| มาตรฐาน   | IEEE 802.11a/b/g/n                        |
| พลังงาน   | สูง                                       |
| ระยะทาง   | ภายในอาคาร                                |
| เหมาะกับ  | กล้อง, สมาร์ตโฮมที่ต้องการแบนด์วิดท์สูง   |
| แนวโน้ม   | ถูกแทนที่ด้วยเทคโนโลยีพลังงานต่ำในงาน IoT |

#  Ethernet

![](Week-01/Images/Ethernet.png)

## 1) แนวคิดหลักของ Ethernet

Ethernet เป็นเทคโนโลยีสื่อสารข้อมูลแบบใช้สายที่ได้รับความนิยมสูงที่สุดในโลกคอมพิวเตอร์และระบบเครือข่ายภายในอาคาร ด้วยความเสถียรสูง ความเร็วสูง และความน่าเชื่อถือ ทำให้ Ethernet ถูกใช้เป็นโครงสร้างพื้นฐานหลักของระบบเครือข่ายมานานหลายทศวรรษ

คุณสมบัติหลักของ Ethernet ได้แก่:
- **เป็นระบบสื่อสารแบบใช้สาย (wired communication)**
- **เสถียรสูงมาก**
- **ความเร็วสูง** (ตั้งแต่ Mbps จนถึงหลาย Gbps)
- **ค่าหน่วงต่ำ (low latency)**
- **ทนต่อสัญญาณรบกวน**
- **เหมาะกับงานที่ต้องการความน่าเชื่อถือสูง**

Ethernet จึงเป็นตัวเลือกหลักในงาน IoT ที่อุปกรณ์อยู่กับที่และต้องการความเสถียร เช่น ระบบอาคารอัจฉริยะ ระบบควบคุมเครื่องจักร และระบบเซิร์ฟเวอร์

## 2) มาตรฐาน IEEE 802.3

Ethernet ทำงานบนข้อกำหนดของมาตรฐาน **IEEE 802.3** ซึ่งกำหนด:
- รูปแบบสัญญาณไฟฟ้า
- ความเร็วในการส่งข้อมูล
- รูปแบบสายเคเบิล
- วิธีการเข้าถึงสื่อ (MAC)
- โครงสร้างเฟรมข้อมูล

มาตรฐานนี้ทำให้ Ethernet สามารถทำงานร่วมกันได้ระหว่างอุปกรณ์จากผู้ผลิตต่าง ๆ และรองรับการพัฒนาในอนาคต เช่น Gigabit Ethernet, 10 Gigabit Ethernet, PoE (Power over Ethernet)

## 3) เหมาะกับระบบ IoT แบบ Stationary

Ethernet เหมาะกับระบบ IoT ที่อุปกรณ์ **อยู่กับที่ (Stationary)** เช่น:
- ระบบควบคุมภายในอาคาร
- ระบบควบคุมเครื่องจักรในโรงงาน
- ระบบเซิร์ฟเวอร์และเกตเวย์ IoT
- ระบบกล้องวงจรปิดแบบใช้สาย
- ระบบควบคุมไฟฟ้าและระบบอาคารอัจฉริยะ (BMS)

เหตุผลคือ:
- ไม่ต้องใช้แบตเตอรี่
- ความเสถียรสูง
- ความเร็วสูง
- ไม่ถูกรบกวนจากสัญญาณวิทยุ
- รองรับการจ่ายไฟผ่านสาย (PoE)

Ethernet จึงเป็น backbone ของระบบ IoT ภายในอาคารและอุตสาหกรรม

## 4) คู่แข่งสำคัญ: Power Line Communication (PLC)

**Power Line Communication (PLC)** เป็นเทคโนโลยีที่ใช้สายไฟฟ้าเป็นสื่อกลางในการส่งข้อมูล ทำให้ไม่ต้องเดินสาย LAN เพิ่มเติม

PLC มีข้อดีคือ:
- ใช้โครงสร้างพื้นฐานเดิม (สายไฟฟ้า)
- ติดตั้งง่ายในอาคารเก่า
- เหมาะกับงานที่ไม่ต้องการเดินสายใหม่

แต่ก็มีข้อจำกัด:
- สัญญาณรบกวนจากอุปกรณ์ไฟฟ้า
- ความเร็วไม่เสถียรเท่า Ethernet
- ไม่เหมาะกับงานที่ต้องการความน่าเชื่อถือสูงมาก

ดังนั้น PLC เป็นคู่แข่งของ Ethernet ในบางสถานการณ์ แต่ Ethernet ยังคงเป็นมาตรฐานหลักในงาน IoT ที่ต้องการความเสถียรสูง

#  สรุป Ethernet

Ethernet เป็นเทคโนโลยีสื่อสารแบบใช้สายที่มีความเสถียรสูง ความเร็วสูง และเหมาะกับงาน IoT ที่อุปกรณ์อยู่กับที่ เช่นภายในอาคารหรือโรงงาน โดยทำงานบนมาตรฐาน IEEE 802.3 และมีคู่แข่งสำคัญคือ Power Line Communication (PLC)

| คุณสมบัติ  | รายละเอียด                    |
| ---------- | ----------------------------- |
| ประเภท     | Wired communication           |
| มาตรฐาน    | IEEE 802.3                    |
| ความเสถียร | สูงมาก                        |
| ความเร็ว   | สูง (Mbps–Gbps)               |
| เหมาะกับ   | IoT แบบ stationary ภายในอาคาร |
| คู่แข่ง    | PLC                           |
